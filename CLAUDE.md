# Project Rules

- 日本語で対応すること
- DESIGN.md を必ず最初に読むこと
- エンジニアリング用語は避け、わかりやすく説明すること
- コードの変更後はブラウザで確認できるように案内すること
- 作業を始める前に、やることを箇条書きで洗い出して提示し、許可を取ってから作業に入ること
- パターンやバリエーションを作成・変更したら、以下を必ずセットで実行すること：
  1. `screens.ts` の画面定義・遷移（linksTo）を更新する
  2. `npm run capture-screens` でスクリーンショットを全画面撮り直す
  3. Compareパネルと画面マップ（/map）が最新状態になっていることを確認する
- ファイルに何かを記載・追記する際は、なぜそのファイルが適切なのか背景を伝えること
- 会話の中で新しいルールや要件が決まったら、「該当ファイルに記載しましょうか？」と積極的に提案すること
- スキルファイル（SKILL.md, SPEC_GUIDE.md, SCREEN_MAP_GUIDE.md）を更新する際は、`~/.claude/skills/vibe-design/` とプロジェクト内 `skills/` の両方を同じ内容に更新すること
